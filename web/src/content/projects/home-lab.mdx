Everyone has opinions about local AI. I wanted receipts. So the lab exists to answer one question with evidence: how much of a modern AI workflow can you run on hardware you own, with no per-token bill and no data leaving the house?

The host is a Windows 11 Pro workstation with 20GB of VRAM, enough to run genuinely useful models, not just demos. Everything interesting happens in Docker: a curated set of containers with security settings tuned so the whole environment is isolated from the rest of the network. Tools get exactly the access they need and nothing else. It's a home lab, but I run it like infrastructure, because sloppy habits at home become sloppy habits everywhere.

At the center is **Ollama**, serving local models that my custom scripts run against: coding experiments, text pipelines, and media generation workflows for images and video. Around it sits the supporting cast that makes local LLMs actually *useful* rather than a toy you poke at: **N8N** for orchestration, plus the glue scripts that turn "a model that can answer" into "an automation that does."

The honest tradeoffs, since that's the point: local models are slower than frontier APIs and you feel the VRAM ceiling on the big ones. But the flip side is real: zero marginal cost per experiment, total data privacy, and the freedom to let automations run around the clock without watching a meter. For a growing class of workloads, that trade wins.

This is also where I test the thesis I keep bringing to client work: AI is only as capable as the foundation under it. The lab is small, but the discipline (isolation, least privilege, observable workflows) is the same one I'd hold an enterprise to.
